A note on missing values. The EDA above shows colSums(is.na(train)) returns zero for every column, and a check of the character columns finds no disguised missingness (no "", "?", "NA", "NULL", or "Unknown" placeholders). The Kaggle Car-Insurance dataset is therefore complete, no imputation is required. The pre-processing effort here instead targets (1) feature engineering of text-encoded specs, (2) consistent encoding of categorical/binary variables, and (3) removal of redundant, highly correlated predictors.

Important data-leakage note. The Kaggle test.csv does not contain the is_claim label. It therefore cannot be used to measure model performance. All training, validation, and test sets in this report are carved from the labeled train.csv (58,592 rows). test.csv is reserved only for an optional Kaggle submission.

Correlated-feature handling

The vehicle-specification predictors (engine size, dimensions, weight) are physically related and highly collinear, which destabilizes linear models. We visualize the numeric correlation structure and drop near-redundant predictors using caret::findCorrelation (|r| > 0.90).

num_pred <- clean %>% select(where(is.numeric))
cormat   <- cor(num_pred)

corrplot(cormat, method = "color", type = "upper",
         tl.cex = 0.6, tl.col = "black", order = "hclust",
         title = "Correlation of Numeric Predictors", mar = c(0,0,1,0))
high_corr <- findCorrelation(cormat, cutoff = 0.90, names = TRUE)

Data Splitting

Because the Kaggle test.csv is unlabeled, we partition the labeled data into a stratified 60/20/20 train/validation/test split. Stratification on is_claim preserves the ~6.4% claim rate in every partition. The training set is used with 5-fold cross-validation for hyperparameter tuning; the validation set is used to compare model families and select a finalist; the test set is touched once to report an unbiased estimate of generalization performance.

Model Strategies

Primary research question. Can policy- and vehicle-level attributes available at underwriting predict whether a policyholder will file a claim in the next six months? This is a binary classification problem framed to support proactive risk pricing and intervention.

Secondary questions. (1) Which predictors carry the most signal for claim risk? (2) Do flexible non-linear learners materially outperform interpretable linear baselines on this data?

The central modeling challenge is severe class imbalance (6.4% positive). Two design choices address it:

  • Resampling: down-sampling the majority class inside each cross-validation fold (sampling = "down" in trainControl), so the resampled performance estimates remain honest and the learners are not overwhelmed by non-claimers.
  • Evaluation metrics: overall accuracy is misleading here (a trivial “no claim” model scores 93.6%). We instead optimize and report ROC-AUC as the primary tuning metric, alongside sensitivity (recall of claims), specificity, balanced accuracy, and F1.

Candidate models span the interpretability–flexibility spectrum, mirroring the Applied Predictive Modeling (Kuhn & Johnson) case studies:

Family Method Rationale
Linear / interpretable Logistic regression, penalized logistic (glmnet), PLS-DA Transparent baselines; handle correlated predictors (glmnet)
Non-linear MARS (earth) Captures hinge/interaction effects while staying interpretable
Tree-based CART, Random Forest (ranger), Gradient Boosting (gbm) Capture complex interactions; robust to mixed-type, correlated predictors

All models are tuned via 5-fold cross-validation on the training set with ROC-AUC as the selection metric.

Model Building and Hyperparameter Tuning

ctrl <- trainControl(method = "cv", number = 5,
                     classProbs = TRUE,
                     summaryFunction = twoClassSummary,
                     sampling = "down",
                     savePredictions = "final")

fit <- function(method, ...) {
  train(is_claim ~ ., data = train_df, method = method,
        metric = "ROC", trControl = ctrl, ...)
}

set.seed(503)
m_glm    <- fit("glm", family = "binomial")
m_glmnet <- fit("glmnet", tuneLength = 5)
m_pls    <- fit("pls", tuneLength = 5)
m_mars   <- fit("earth", tuneLength = 5)
m_rpart  <- fit("rpart", tuneLength = 8)
m_rf     <- fit("ranger", tuneLength = 4, num.trees = 400, importance = "impurity")
m_gbm    <- fit("gbm", verbose = FALSE,
                tuneGrid = expand.grid(n.trees = c(150, 300, 500),
                                       interaction.depth = c(2, 4, 6),
                                       shrinkage = 0.05, n.minobsinnode = 10))

models <- list(Logistic = m_glm, Penalized = m_glmnet, PLS = m_pls,
               MARS = m_mars, Tree = m_rpart, RandomForest = m_rf, GBM = m_gbm)
